# Предварительные требования
Перед началом работы над проектом необходимо установить следующие библиотеки:
- Node (>= v18.17)
- Yarn (v1 or v2+)
- Git

# Инструкция
1. Клонируйте репозиторий и установите зависимости.  
    `git clone https://github.com/scaffold-eth/scaffold-eth-2.git`  
    `cd <project_name>`  
    `yarn install`

2. Удалить файлы.
    - packages/hardhat/contracts/YourContract.sol (дефолтный контракт)
    - packages/hardhat/deploy/00_deploy_your_contract.ts (дефолтный деплой контракта)
    - packages/hardhat/test/YourContract.ts (дефолтныe тесты контракта)  


3. Добавляете новые файлы в проект из яндекс диска. 
    **Папка бэкенд**
    - packages/hardhat/contracts/Voting.sol (контракт)
    - packages/hardhat/deploy/VotingDeploy.ts (деплой контракта)
    - packages/hardhat/test/VotingTest.ts (тесты контракта)  

    **Папка фронтенд**
    - packages/nextjs/components/CreateForm.tsx
    - packages/nextjs/components/Poll.tsx
    - packages/nextjs/components/PollForm.tsx
    - packages/nextjs/components/VoteForm.tsx
    - packages/nextjs/styles/components.css

4. Обновить файлы
    **Папка фронтенд**
    - packages/nextjs/app/page.tsx 

5. Запустите сеть в терминале.  
`yarn chain`

6. Откройте второй терминал и выполните деплой контракта.  
`yarn deploy`

7. Откройте третьй терминал и запустите фронтенд.  
`yarn start`

8. Перейдите в пользовательский интерфейс приложения по http://localhost:3000/ и сделайте скриншот. Добавьте этот скриншот в отчет.  
Опционально можно потыкать функции в интерфейсе, чтобы на скрине было видно работоспособность приложения. 

9. Открыть новый терминал и запустить тесты  
`yarn test`

10. Сделать скриншот пройденных тестов и добавить его в отчет.

11. Сделать коммит с добавленными файлами и запушить в свой удаленный репозиторий

12. Добавить ссылку на гит в отчет

13. Обновить номера страниц в содержании отчета
